# Maintainers

This document lists the maintainers of the Substrait Compliance Framework project.

## Current Maintainers

Maintainers are responsible for the overall direction, health, and sustainability of the project. They review contributions, manage releases, and represent the project in the Substrait community.

## Maintainer Responsibilities

Maintainers are expected to:

1. **Code Review**: Review and provide feedback on pull requests
2. **Issue Triage**: Label, prioritize, and respond to issues
3. **Release Management**: Coordinate and execute releases
4. **Community Engagement**: Participate in discussions and community calls
5. **Security**: Respond to security vulnerabilities promptly
6. **Documentation**: Ensure documentation stays current
7. **Mentorship**: Guide new contributors and committers
8. **Governance**: Participate in governance decisions

## Maintainer Meetings

- **Frequency**: Bi-weekly
- **Format**: Video call (recorded and shared publicly)
- **Agenda**: Posted in GitHub Discussions 48 hours before meeting
- **Notes**: Published in GitHub Discussions after meeting

## Becoming a Maintainer

See [GOVERNANCE.md](GOVERNANCE.md) for the process of becoming a maintainer.

**Requirements:**
- Serve as a Committer for 6+ months
- Demonstrate sustained, high-quality contributions
- Show leadership in technical direction
- Commitment to project sustainability
- Nomination by existing Maintainer
- Approval by majority of current Maintainers
