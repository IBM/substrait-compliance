# Test Enhancement Workflow Guide

Complete guide for enhancing Substrait function tests using Claude AI quality analysis.

## Overview

The test enhancement workflow consists of three main steps:

1. **Quality Check** - Analyze test files for gaps and issues
2. **Enhancement** - Generate improved tests based on analysis
3. **Comparison** - Compare original vs enhanced tests

## Step 1: Quality Check

Analyze your test files to identify gaps and issues.

### Check Single Category

```bash
python scripts/quality_checker.py arithmetic
```

### Check Specific File

```bash
python scripts/quality_checker.py --file test-suites/functions/arithmetic/add.test
```

### Check All Categories

```bash
python scripts/batch_quality_check.py
```

### Output

Quality reports are saved to `quality_reports/`:
- `<category>_quality.json` - Detailed analysis per category
- `summary.json` - Overall summary
- `QUALITY_REPORT.md` - Human-readable report

### Example Quality Report

```json
{
  "suite_name": "add",
  "test_count": 27,
  "analysis": {
    "overall_score": 7.5,
    "issues": [
      "Missing test for negative number addition",
      "No boundary value test for i64 max"
    ],
    "missing_tests": [
      "add(-5::i32, -3::i32) = -8::i32",
      "add(9223372036854775807::i64, 0::i64) = 9223372036854775807::i64"
    ],
    "recommendations": [
      "Add more boundary value tests",
      "Include tests for all numeric types"
    ]
  }

```

## Step 2: Test Enhancement

Generate improved test files based on quality analysis.

### Enhance Single Category

```bash
python scripts/test_enhancer.py arithmetic
```

This will:
1. Run quality check on all files in the category
2. Use Claude to generate missing test cases
3. Save enhanced tests to `test-suites-enhanced/functions/arithmetic/`

### Enhance Specific File

```bash
python scripts/test_enhancer.py --file test-suites/functions/arithmetic/add.test
```

### Custom Output Directory

```bash
python scripts/test_enhancer.py arithmetic --output my-enhanced-tests/functions
```

### Output

Enhanced test files are saved with:
- Original test cases preserved
- New test cases added at the end
- Clear markers showing AI-generated content
- Enhancement metadata in comments

## Best Practices

### 1. Review Before Committing

Always review enhanced tests before committing:

```bash
# View diff
diff test-suites/functions/arithmetic/add.test \
     test-suites-enhanced/functions/arithmetic/add.test

# Or use git diff
git diff --no-index test-suites/functions/arithmetic/add.test \
                    test-suites-enhanced/functions/arithmetic/add.test
```

### 2. Validate Enhanced Tests

Run the enhanced tests through your compliance framework:

```python
from substrait_compliance.function_test_parser import FunctionTestParser
from substrait_compliance.runner import ComplianceRunner

parser = FunctionTestParser()
suite = parser.parse_file("test-suites-enhanced/functions/arithmetic/add.test")

# Validate with your engine
runner = ComplianceRunner(your_engine)
report = runner.run_test_suite(suite)
print(f"Pass Rate: {report.get_pass_rate()}%")
```

### 3. Iterative Enhancement

Enhance in iterations:

```bash
# First iteration
python scripts/test_enhancer.py arithmetic

# Review and validate
python scripts/test_comparator.py arithmetic --html

# If needed, run quality check on enhanced tests
python scripts/quality_checker.py --file test-suites-enhanced/functions/arithmetic/add.test

# Second iteration if score is still low
# (manually edit or re-enhance)
```

### 4. Merge Selectively

Don't blindly merge all enhanced tests. Review and select:

1. **High-value additions**: Tests that fill real gaps
2. **Correct tests**: Verify mathematical/logical correctness
3. **Relevant tests**: Ensure tests match function behavior
4. **Non-duplicate tests**: Avoid redundant test cases

### 5. Document Changes

Add comments explaining why tests were added:

```
# ai_generated: Added based on quality analysis
# Rationale: Missing boundary value tests for i64 type
add(9223372036854775807::i64, 0::i64) = 9223372036854775807::i64
add(-9223372036854775808::i64, 0::i64) = -9223372036854775808::i64
```

## Configuration

### Customize Enhancement Behavior

Edit `scripts/quality_config.yaml`:

```yaml
enhancement:
  # Number of tests to generate per category
  tests_per_category: 5-10
  
  # Temperature for test generation (0.0-1.0)
  # Lower = more conservative, Higher = more creative
  temperature: 0.4
  
  # Focus areas
  focus:
    - boundary_values
    - null_handling
    - overflow_scenarios
    - negative_numbers
    - edge_cases
```

Use different models for different purposes:

```bash
# Fast enhancement with Haiku

# High-quality enhancement with Opus
export MODEL="aws/claude-opus-4-6"

```

## Troubleshooting

### Issue: No Tests Generated

**Cause**: Quality score is already high (>9.0)

**Solution**: Tests are already comprehensive. Manual review recommended.

### Issue: Generated Tests Are Incorrect

**Cause**: Model hallucination or misunderstanding

**Solution**: 
1. Review and manually correct
2. Lower temperature in config
3. Provide more context in quality check

### Issue: Too Many Duplicate Tests

**Cause**: Model generating variations of existing tests

**Solution**:
1. Use test_comparator to identify duplicates
2. Manually remove redundant tests
3. Adjust enhancement prompt

### Issue: Enhanced File Parse Errors

**Cause**: Generated tests don't match format

**Solution**:
1. Check test file syntax
2. Validate with parser:
   ```python
   parser.parse_file("test-suites-enhanced/functions/arithmetic/add.test")
   ```
3. Manually fix format issues

## Integration with CI/CD

### GitHub Actions Example

```yaml
name: Test Enhancement

on:
  schedule:
    - cron: '0 0 * * 0'  # Weekly on Sunday

jobs:
  enhance-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      
      - name: Install dependencies
        run: pip install -r sdk/python/requirements.txt
      
      - name: Run quality check
        run: python scripts/batch_quality_check.py
      
      - name: Enhance tests
        run: |
          for category in arithmetic boolean string; do
            python scripts/test_enhancer.py $category
          done
      
      - name: Generate comparison reports
        run: |
          for category in arithmetic boolean string; do
            python scripts/test_comparator.py $category --html
          done
      
      - name: Upload reports
        uses: actions/upload-artifact@v3
        with:
          name: enhancement-reports
          path: quality_reports/
      
      - name: Create PR with enhancements
        uses: peter-evans/create-pull-request@v5
        with:
          title: 'AI-Enhanced Test Cases'
          body: 'Automated test enhancements based on quality analysis'
          branch: test-enhancements
```
